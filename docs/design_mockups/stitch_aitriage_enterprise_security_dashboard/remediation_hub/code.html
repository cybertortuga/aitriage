AI RATIONALE

The original code used a signed integer (int) for val and lacked upper-bound checks against SIZE_MAX prior to multiplication. This allows an attacker to craft a payload that overflows the multiplication val * sizeof(uint32_t), resulting in a small allocation followed by a massive heap write (CVE-2023-4863). The patch changes the type to uint32_t and adds an explicit bounds check to ensure the multiplication cannot overflow SIZE_MAX.
